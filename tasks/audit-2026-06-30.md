## Follow-up: GRANT-Permissions (2026-06-30)

**Symptom (Production-Hit):**
- `Datenbankfehler beim Anlegen: permission denied for table group_shared_expenses`
- `Datenbankfehler beim Anlegen des Ausflugs: permission denied for table group_trips`

**Root-Cause:** Supabase auto-grantet `anon`/`authenticated`/`service_role` Permissions
nur auf Tabellen die via Dashboard-UI angelegt werden — nicht für Tabellen aus dem
SQL Editor. Die 10 neuen Tabellen aus `2026-06-29_groups_expansion.sql` waren
deshalb für Hyperdrive-Connections unzugänglich.

**Fix:** Neue Migration `seeds/migrations/2026-06-30_grant_permissions.sql` muss
ausgeführt werden. Sie:
- `GRANT ALL` auf alle 10 neuen Tabellen
- `GRANT ALL ON ALL SEQUENCES` (für SERIAL ID columns)
- `GRANT EXECUTE` auf alle 5 RPC-Funktionen
- Setzt `ALTER DEFAULT PRIVILEGES` damit zukünftige Tabellen automatisch GRANTs erhalten

Idempotent — kann beliebig oft ausgeführt werden.

**User-Action:**
1. Migration `seeds/migrations/2026-06-30_grant_permissions.sql` im Supabase SQL-Editor ausführen.
2. Testen: Gruppenausgabe + Ausflug anlegen — sollte jetzt funktionieren.

## Follow-up: Sparquote-Anzeige

**Symptom:** Bei deutlich höheren Ausgaben als Einnahmen zeigte das Dashboard
"0.0%" Sparquote statt der echten negativen Rate.

**Root-Cause:** `Math.max(0, ...)` clampte das Ergebnis auf 0.

**Fix:** Clamp entfernt; negative Sparquote wird jetzt rot dargestellt mit
Hinweis "Ausgaben übersteigen Einnahmen". UX-Mehrwert: User sieht ehrlich
seine Finanzlage.
